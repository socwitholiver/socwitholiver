### 🧠 &nbsp;About me &nbsp;<sub>· &nbsp;(OJV® — 01)</sub>

I'm **Oliver** — an IT student working the seam where **defensive security meets data analytics**. I pair blue-team investigative thinking with the explore → model → visualise loop, because a security alert and a messy dataset are the same job: *something is making noise, and someone needs to know what it means.*

```yaml
role:      SOC analyst in training  ·  data analyst
focus:     blue-team fundamentals · SIEM & log analysis · malware behaviour
building:  hands-on security labs, not just slide decks
mindset:   "what could go wrong?"  →  gather evidence  →  make it defensible
based_in:  Nairobi, Kenya (GMT+3) · remote-ready
```

> Security isn't about tools — it's about mindset, curiosity, and asking *"what could go wrong?"*

<br/>

<!-- ======================================================================= -->
### 🔁 &nbsp;How I work &nbsp;<sub>· &nbsp;(OJV® — 02)</sub>

The same three moves, whether it's an alert or a dataset.

| `01` — Investigate | `02` — Analyse | `03` — Communicate |
| :--- | :--- | :--- |
| Pull the raw evidence — logs, packets, the unclean source table — and get to facts I can defend line by line. | Find the pattern, and rule out the ones that aren't there. Separate anomalous from merely unusual. | Turn the finding into something a stakeholder reads in 30 seconds and acts on — a dashboard, a report, a next step. |
